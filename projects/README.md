# projects

このフォルダは、codex-company自身で作成・検証するアプリや継続案件を置く場所です。

## 新規アプリ作成ルール

新しいアプリや継続案件を作成する場合は、以下の構成にします。

```text
projects/<アプリ名>/README.md
```

最初に `README.md` を作成し、アプリ概要、目的、想定ユーザー、主な機能、技術構成を書いてから実装ファイルを追加します。

## 例

```text
projects/my-app/README.md
projects/receipt-fridge-ai/README.md
projects/gameapp/README.md
```

## 注意

- アプリ名が決まっていない場合は、勝手にフォルダを作らず確認します。
- プロジェクト直下にアプリ用ファイルを直接置きません。
- README.mdなしで実装ファイルだけを作り始めません。
